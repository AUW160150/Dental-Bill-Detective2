Integration 1 of 5 · PHI Protection
🔐 Civic

Identity
Verification

Before any dental bill data is stored or uploaded, the user's identity is verified via Civic's Secure Identity Platform. No PHI moves without a valid verification token.

Gate role: Acts as the HIPAA-safe PHI door — nothing gets cached in Redis or indexed in Contextual AI until Civic says the user is verified.
# civic_auth.py from civic_auth import verify_user # Called before EVERY data operation verification = verify_user( civic_api_key=CIVIC_API_KEY, token=user_civic_token ) if not verification["verified"]: raise PermissionError( "PHI processing blocked" ) user_id = verification["user_id"] # ↑ stable hash — safe Redis key

Integration 2 of 5 · Live Pricing Data
⚡ Apify

Real-Time
Benchmark Scraping

Apify's Puppeteer-based web-scraper actor fetches live pricing data from FairHealth consumer estimates and CMS Medicare fee schedules — the same data insurers use internally.

Data role: Converts opaque CDT codes into a fair-market dollar range. Without this, there's nothing to compare billed amounts against.
FairHealth Consumer CMS Fee Schedule ADA CDT Descriptions
# scrape.py from apify_client import ApifyClient client = ApifyClient(APIFY_API_TOKEN) # Puppeteer scraper — JS-rendered pages run = client.actor( "apify/web-scraper" ).call(run_input={ "startUrls": [{ "url": "https://fairhealthconsumer.org" }], "maxCrawlingDepth": 1, }) for item in client.dataset( run["defaultDatasetId"] ).iterate_items(): # → { code: "D2393", p80: 265 }

Fetched benchmarks for 6 unique CDT codes in ~3.5 seconds. D2393 80th pct = $265, D8680 = $1,050, D8090 = $6,800.
Integration 3 of 5 · Multi-Hop RAG Audit
🧠 Contextual AI

RAG-Powered
Bill Auditing

The bill PDF and FairHealth benchmark data are uploaded to a Contextual AI datastore. The RAG agent performs multi-hop reasoning across both documents to audit each CDT code.

Reasoning role: Is each code billed correctly? Duplicate charges? Unbundling? Upcoding? It cross-references the bill against pricing benchmarks simultaneously.
# bill_analyzer.py from contextual import ContextualAI client = ContextualAI( api_key=CONTEXTUAL_API_KEY ) # Upload bill to datastore requests.post( f"/v1/datastores/{DATASTORE_ID}/documents", files={"file": (filename, pdf_bytes)} ) # IMPORTANT: use .create() not .query() resp = client.agents.query.create( agent_id=AGENT_ID, messages=[{"role": "user", "content": audit_prompt}] ) answer = resp.message.content
⚠ SDK gotcha:
Use client.agents.query.create() — NOT client.agents.query(). The latter raises QueryResource object is not callable.
